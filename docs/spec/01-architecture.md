## 6. 并发模型

### 6.1 并行管道架构

```python
# ParallelPipeline 使用 asyncio.Queue 实现生产者-消费者模式

class ParallelPipeline:
    def __init__(self):
        self.download_queue = asyncio.Queue(maxsize=10)  # 下载队列
        self.upload_queue = asyncio.Queue(maxsize=20)    # 上传队列

    async def run(self):
        tasks = [
            # 1 个下载生产者
            self._download_producer(slugs, progress, task_id),
            # 1 个调度器
            self._download_to_upload_dispatcher(),
            # N 个上传消费者（每个上传器 M 个 worker）
            self._upload_consumer(uploader, progress, task_id),
            # ... more consumers
        ]
        await asyncio.gather(*tasks)
```

### 6.2 背压控制

```
┌─────────────────────────────────────────────────────────────┐
│                     Backpressure Control                     │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  1. Download Queue Size Limit (10)                          │
│     - 防止下载过快导致内存问题                                │
│                                                             │
│  2. Upload Queue Size Limit (20)                            │
│     - 平衡下载和上传速度                                      │
│                                                             │
│  3. Max Pending Uploads (50)                                │
│     - 限制等待上传的文件数量                                  │

│                                                             │
│  4. Min Disk Free (5GB)                                     │

│     - 等待上传完成释放空间                                    │
│                                                             │
│  5. Pause File (.pause_downloads)                           │
│     - 外部监控脚本可创建暂停文件                              │

│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 7. 错误处理策略

### 7.1 重试机制

| 操作 | 重试策略 | 最大重试 |
|------|---------|---------|
| HTTP 请求 | 指数退避 | 3 次 |
| 视频下载 | 标记失败，后续重试 | - |
| 上传失败 | 记录错误，保留文件 | - |

### 7.2 错误恢复

```
┌─────────────────────────────────────────────────────────────┐
│                     Error Recovery                           │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  1. 下载失败                                                 │
│     - 标记状态为 'failed'                                    │
│     - 记录 error_message                                     │
│     - 删除部分下载文件                                       │
│     - 可通过 --retry 重新尝试                                │
│                                                             │
│  2. 上传失败                                                 │
│     - 记录到日志                                             │
│     - 保留本地文件（不删除）                                  │
│     - 统计 upload_failed                                    │
│                                                             │
│  3. 管道中断                                                 │
│     - 通过 archive 文件记录已下载                            │
│     - 数据库记录进度状态                                     │
│     - 重启后从断点继续                                       │
│                                                             │
│  4. 数据库错误                                               │
│     - 使用 replace=True 避免主键冲突                         │
│     - 事务保证数据一致性                                     │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 8. 性能考虑

### 8.1 资源使用

| 资源 | 使用方式 | 限制 |
|------|---------|------|
| 内存 | asyncio 并发 | 队列大小限制 |
| 磁盘 | 临时文件存储 | 最小剩余空间检查 |
| 网络 | HTTP 连接池 | 最大连接数配置 |
| CPU | 下载/上传并行 | 线程池执行器 |

### 8.2 优化点

1. **异步 I/O**: 使用 asyncio 实现非阻塞并发
2. **连接复用**: httpx 连接池
3. **流式下载**: 大文件分块下载
4. **无编码分割**: ffmpeg -c copy 快速分割
5. **延迟控制**: 随机延迟避免被封
